Fibonacci & Memoization

What?

  • 0 1 1 2 3 5 8
  • 0 f(0)=1 f(1)=1 f(2)=3 f(3)=3 f(4)=5 f(5)=8
  • f(n) = f(n-1) + f(n-2)
  • Recursive
  • Memoization is caching the result of old computations needed by new computation

Why?

  • Using recursion to compute fibonacci value for a given number has time complexity of O(2^n) - Exponential time
  • By storing values in an array of previous fibonacci value, recursion is avoided if the value is available. This reduces time taken for computation to O(n)

How?

int [ ] mem = new int[1001];
public int fib(n){
if (n== 0)
return 1;
if (n== 1)
return 1;
if (mem[n] == 0){
mem[n] = fib(n-1) + fib(n-2);
}
return mem[n];
}

Time Complexity

  • Linear time (using memoization)
