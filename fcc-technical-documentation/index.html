Object Oriented Programming

Modeling real world objects in programs is the core philosophy of object oriented programming. Characteristics and functionality makes each class of object unique. For e.g. Book is an object. It has the following characteristics

  • Title
  • ISBN - a unique identifier
  • Author
  • No. of pages
  • Genere
			    

public class Book {
	private String title;
	private String isbn;
	private String author;
	private String genre;
	private String noofpages;
}

Encapsulation

Protecting the characteristics or state of an object after its creation is called encapsulation. This is important for creating reliable software programs that are immune to inadvertent misuse of object state. For eg.: After creating a Book object, Angels and Demons, from the class Book described above to be listed in the product search page of our website, its characteristics should not be altered in any way. This is accomplished by making the characteristics (variables) private.
